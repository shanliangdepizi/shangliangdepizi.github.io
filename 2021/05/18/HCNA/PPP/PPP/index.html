PPP协议应用

简介

  1. 一种点对点协议
  2. 用于全双工的同步、异步链路上进行进行点对点的数据传输

名称 认证 认证错误检测 错误恢复 支持同步或异步链路

PPP组件

名称 作用
LCP Link Control Protocol
链路控制协议		 用来建立、拆除和监控PPP数据链路以及认证
NCP Network Control Protocol
网络层控制协议		 用于对不同的网络层协议进行连接建立和参数协商

PPP认证概述

认证模式

模式 备注
PAP PassWord Authentication Protocol
密码认证协议		 明文方式直接发送密码
二次握手机制
可以做无限次尝试（暴力破解）
只在链路建立的时候验证，链路建立之后不再验证
CHAP Chanllege Handshake Authentication Protocol
挑战/质询握手认证机制		 以MD5加密信息
三次握手机制
由认证方发起
避免暴力破解
链路建立之后，有再次认证检测机制
在企业网中，远程接入环境中使用

PAP

  1. 图示

  2. 认证流程

    • 被认证方发送 Authenticate-Request包，包内是明文发送的用户名密码
    • 认证方检查被认证方发送的信息，并回复给被认证方认证成功（Authenticate-ACK）或者失败（Authenticate-Nak）

CHAP

  1. 图示

  2. 认证流程

    • 认证方发起Challenge，将信息MD5加密
    • 被认证方将收到的Challenge信息Respose给认证方
    • 认证方将收到的信息进行比对，回复给被认证方Success后者Failure
